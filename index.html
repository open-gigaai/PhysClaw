Method

Two timescales, a strict division of labor

Within a session, the system changes its behavior through text alone. A single instruction ("learn to tidy the tabletop") is expanded into a four-part plan: a collection routine, a collection success criterion, a reset routine, and a reset success criterion. Collection then runs as a closed loop — execute, let the scene settle, query a VLM verifier for a confidence score. A phase passes at threshold θ and retries up to N times; exhausting retries halts the arm into the Alert state. The pair (θ, N) makes the boundary between "keep going" and "ask a human" an explicit, tunable hyperparameter rather than an accident of implementation.

Between sessions, the model changes its weights through data alone. Trajectories certified under the operator-corrected criteria fine-tune the underlying VLA policy. Human language reaches the weights only indirectly — through the data it certifies — and the improved policy returns to collection with fewer failures and fewer alerts.

Corrective Memory

Each operator sentence is parsed into a structured entry — trigger · correction · scope · source utterance — and stored as legible Markdown the operator can read, audit, and delete. Because corrections live as text at the orchestration layer, they modify neither VLA weights nor the perception stack, and remain valid when the underlying policy is swapped out. Round-scoped nudges ("nudge it slightly left this time") are applied once and discarded; conditioned rules ("for flat, low-profile objects, grasp 1 cm below the proposed height") persist and generalize to every matching object.

Why the reset is verified separately

A failed grasp is self-evident; a failed reset is silent. Unchecked, the scene drifts a little further from its intended state every round until no grasp can succeed. Verifying reset against its own criterion is what lets a multi-hour run pause before it degrades unnoticed.

Language-guided correction · §4.2

Sentences that repair both perception and action

The channel presupposes nothing about what feedback contains. The same parser handles a success criterion that is too strict, an object that should be skipped, and a grasp that needs a condition.

Closing the loop · §4.3

The data is good enough to beat teleoperation

Policies fine-tuned from the same VLA checkpoint, evaluated over 20 blind physical trials per task — no agent, no verifier, no memory lookup, no human. Zero-shot average:

Pre-trained base17.5%
Budget-matched teleop (4.8 human-min)32.5%
Scripted collector data42.5%
Full human teleop (46 human-min)82.5%
PhysGnosis data (4.8 human-min)87.5%

Equal human time alone is not enough — the budget-matched teleop set is simply too small. PhysGnosis occupies the useful operating point: 50 valid demos with action coverage close to full teleoperation (action std. 0.20 vs 0.22, diversity 0.35 vs 0.38), at roughly a tenth of the runtime human minutes. A policy trained on one run becomes the collector for the next; the corrections that made early interventions persistent keep applying; human involvement diminishes as the flywheel turns.

Abstract

Abstract

Autonomous data collection governs both the volume and quality of real-world trajectories available for manipulation policy learning. Existing pipelines reduce human effort through self-resetting mechanisms, VLM-based verification, or language-guided correction — but the knowledge produced by each intervention is discarded after the current episode, so recurrent failure modes demand repeated attention and the cost of oversight grows with session duration. We present PhysGnosis, a human-robot symbiotic agentic system in which the robot improves under natural-language guidance. The operator supplies corrective knowledge in natural language, and every correction is retained for reuse in all subsequent rounds. As this knowledge accumulates, the system resolves recurring failures on its own and demands progressively less human attention. On a real-robot pick-and-place testbed, PhysGnosis matches the collection success rate of full human teleoperation while reducing the human-time ratio below X% — final number. Language corrections repair both verifier criteria and execution strategies, raising verifier accuracy from 40.0% to 100.0% and collection success from 25.0% to 65.0% across object categories. Closing the loop to deployment, policies fine-tuned on PhysGnosis-collected data outperform the base policy by 70.0 pts (derived from Table 5) in zero-shot success rate, comparable to policies trained on full teleoperation data at a fraction of the runtime and human cost.
